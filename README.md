# P2P-Mediator-Server
 Peer-to-Peer connection coordinator server on C++ with Boost

## План реализации проекта:

1.**Подготовка проекта и проектирование архитектуры**

- Продумать общую архитектуру системы (сервер-посредник и клиенты).
- Подключить необходимые библиотеки и фреймворки (UDT, HTTP-библиотека) с помощью CMake.
- Определить формат обмена сообщениями между клиентом и сервером (регистрация, запрос на соединение, передача адресов).

2.**Разработка прототипа приложения** 
- Клиент
  - Реализовать прототип клиента, подключающегося к серверу-посреднику с использованием UDT.
  - Передавать серверу регистрационные данные: ID, публичный IP-адрес и порт.
- Сервер
  - Реализовать прототип сервера, принимающего входящие соединения по UDT.
  - Обеспечить регистрацию и хранение данных клиентов (ID, публичный адрес, порт).

3.**Расширение функционала**
- Реализовать обмен данными между клиентами через сервер (обмен адресами и портами по UDT).
- Добавить функционал установки P2P-соединения между клиентами и реализовать базовую HTTP-коммуникацию по установленному каналу.

**P.S. План не полный и может быть дополнен по ходу разработки**
